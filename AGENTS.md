# Rules for Future Codex Sessions

本仓库的负责人是学生本人。Codex 负责协助工程执行、验证和记录，不替代研究判断。

1. 未经用户明确确认，不得实现完整论文方法、teacher/student model 或 Knowledge Distillation loss。
2. 每次修改前，先用简短中文解释目的与影响。
3. 每次只完成一个边界清楚、可以验证的任务。
4. 不得伪造、补写或推测实验结果。
5. 不得隐藏 failing tests、warnings 或失败命令。
6. 不得删除 experiment logs、failure logs 或 raw results 来美化结果。
7. 不得自动修改 research question、主要假设或评价标准。
8. 未经确认，不得运行长时间训练或下载 dataset。
9. 所有重要修改必须配有适当测试或明确 verification。
10. 所有 research claims 必须区分“论文声称”与“本项目实际观察”。
11. 不得把 example config 描述为最终实验配置。
12. 不得创建 GitHub remote、公开仓库或上传 secrets、datasets、checkpoints。
